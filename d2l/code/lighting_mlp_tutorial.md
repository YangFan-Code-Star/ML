# 用多层感知机拟合渲染照度

本节参考《动手学深度学习》的写作风格，将“灯光属性到渲染结果亮度”的问题写成一个标准的监督学习回归任务。我们把渲染结果的照度近似定义为灰度图像的中值灰度，并用多层感知机拟合灯光属性与该亮度指标之间的非线性关系。

## 1. 问题背景与任务定义

在真实渲染系统中，主光强度、灯光数量、环境光强度、灯光高度和入射角都会共同影响场景的明暗分布。如果我们希望快速预测一组灯光参数对应的整体亮度，而不是每次都完整渲染，就可以把这个问题近似成一个回归任务。

设输入特征为：

- `intensity_main`：主光强度
- `light_count`：灯光数量
- `ambient_intensity`：环境光强度
- `main_height`：主光高度
- `main_angle`：主光入射角
- `fill_ratio`：辅光占主光比例

输出标签记为 `y`，表示渲染结果转为灰度图后的中值灰度。中值灰度处于 `[0, 1]` 区间，可以作为场景整体照度的稳健近似。

## 2. 生成合成数据集

本实验先不接真实渲染器，而是构造一份符合直觉的合成数据：

- 主光强度与环境光强度越大，整体更亮
- 灯光数量增多时亮度上升，但存在边际递减
- 主光入射角通过余弦项影响有效照度
- 灯光高度偏离合理区域时，亮度会发生衰减
- 辅光比例与主光强度之间存在交互作用
- 最后叠加小幅噪声，并把结果裁剪到 `[0, 1]`

这类关系显然不是简单线性的，因此更适合使用多层感知机。

## 3. 读取数据与小批量迭代

我们生成约 `5000` 条样本，并按 `80%/20%` 划分训练集和测试集。为了让不同量纲的输入更容易训练，需要只用训练集统计量对特征做标准化，而标签保持原始灰度尺度。

实现中优先使用 `d2l.load_array` 构造小批量数据迭代器；如果环境里暂时没有安装 `d2l`，脚本会自动退回到 `torch.utils.data.DataLoader`。

## 4. 定义多层感知机

本节使用一个两层隐藏层的回归网络：

```text
Linear(6, 64) -> ReLU -> Linear(64, 32) -> ReLU -> Linear(32, 1)
```

其中输入维度为 6，对应 6 个灯光属性，输出维度为 1，对应预测的中值灰度。

## 5. 定义损失函数与优化器

由于这是回归任务，我们使用均方误差损失：

\[
\ell(\hat{y}, y) = \frac{1}{n}\sum_{i=1}^{n}(\hat{y}_i - y_i)^2
\]

优化器采用 Adam，它在这类中小规模的表格回归任务上通常收敛较快。

## 6. 训练模型

训练阶段会在每个 epoch 结束后同时计算训练集和测试集的 `MSE`，并额外记录 `MAE`，便于观察误差是否持续下降、模型是否学到了稳定映射。

脚本中已经封装了以下函数接口：

- `synthetic_lighting_data(num_examples)`：生成合成灯光数据
- `load_data(batch_size, test_ratio)`：划分数据、标准化并构造迭代器
- `get_net()`：返回 MLP 模型
- `train(net, train_iter, test_iter, num_epochs, lr)`：训练并记录历史指标
- `evaluate(net, data_iter)`：计算 `MSE` 和 `MAE`

## 7. 可视化训练误差与预测效果

脚本会在 `code/outputs/` 下保存三类图像：

- `feature_scatter.png`：代表性特征与中值灰度的散点图
- `loss_curve.png`：训练集和测试集损失曲线
- `prediction_scatter.png`：预测值与真实值对比散点图

此外，终端中还会打印若干测试样本的真实灰度、预测灰度和绝对误差，方便快速检查回归效果。

## 8. 小结

这个实验把“渲染参数到亮度”的映射抽象成了一个紧凑的回归问题。由于其中包含角度余弦、边际递减、交互项和非线性衰减，多层感知机通常比单纯线性模型更能刻画这种关系。

当测试集误差显著低于标签方差基线时，说明模型已经学到了有效的照度近似映射。后续如果接入真实渲染器，只需要把合成数据模块替换为真实 CSV 或渲染采样结果，其余训练流程基本可以保持不变。

## 9. 练习

1. 删除隐藏层，把网络改成线性回归，比较测试误差。
2. 把 `fill_ratio` 的作用改弱或改强，观察模型收敛行为有什么变化。
3. 加入颜色温度、阴影强度或材质反照率，扩展这个回归任务。
4. 尝试用更少的数据训练，分析数据量对泛化误差的影响。
