# 11.1-11.2 优化与深度学习、凸性

## 一句话总结

**大白话**：优化就是「下山找最低点」。但深度学习真正想要的不是「训练集这座山的谷底」，而是「换座山（新数据）也低」（[[泛化误差|泛化]]）。下山路上有坑：假谷底（局部极小）、平坦的马鞍（鞍点）、又窄又陡的峡谷（病态）。凸函数是「只有一个谷底」的理想地形，虽然真实网络不凸，但凸分析能指导我们设计下山方法。

**严谨说法**：优化的目标是最小化训练损失，但深度学习真正关心的是[[泛化误差|泛化]]；优化面临局部极小、鞍点、病态等挑战；凸函数有「局部最优即全局最优」的良好性质，是分析优化算法的理论起点。

## 本节解决的问题

- 凸集/凸函数为何便于优化？

## 核心概念 / 公式 / API

| 概念 | 含义 |
| --- | --- |
| 优化目标 | 最小化训练损失（代理目标） |
| 泛化目标 | 最小化未见数据期望损失（真实目标） |
| 局部极小 | 邻域最优但非全局 |
| 鞍点 | 梯度为 0 但非极值（高维更常见） |
| 病态 | Hessian 条件数大，收敛慢 |

### 凸性
- 凸集：任意两点连线在集合内。
- 凸函数：$f(\lambda x+(1-\lambda)y)\le\lambda f(x)+(1-\lambda)f(y)$。
- 凸函数**局部最优即全局最优**；深度网络损失非凸，但凸分析仍指导算法设计。
- 詹森不等式：$f(E[X])\le E[f(X)]$（凸函数）。

## 易错点

- 把训练损失降到极低当成目标——可能过拟合，泛化才是目的。
- 高维下鞍点比局部极小更普遍，是优化主要障碍。
- 深度网络非凸，凸理论是近似指导而非严格保证。
