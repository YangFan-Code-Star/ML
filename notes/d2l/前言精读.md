# 前言精读 (Preface)

## 一句话总结

**大白话**：这本书不让你先啃完一堆数学再动手，而是「边写边学」——先跑通一个模型尝到甜头，再回头补原理；每个重要零件都给两个版本：自己手搓一遍 + 调框架一行搞定，让你既懂里子又会用。

**严谨说法**：深度学习近年取得突破性进展，本书用「代码 + 数学 + 可执行文本」三位一体的形式，让你在**实践中即时学习**，并通过**从零实现 + 简洁 API 双版本**真正理解每个组件为什么这样设计。

## 本节解决的问题

- 为什么要有这样一本书？市面上的资料要么只有代码缺解释，要么只有理论缺实现，要么藏在付费墙后。
- 这本书想同时满足：(1) 内容最新；(2) 覆盖现代深度学习全领域且有技术深度；(3) 既有干净可运行代码，又有高质量阐述。
- 应用深度学习到底需要哪些能力？见 [[概念-四要素]]。
- 应该用什么方式学？见 [[概念-在实践中学习]]。

## 核心思想（逐节梳理）

### 1. 一种结合了代码、数学和 HTML 的媒介
- 公式用 LaTeX、代码用 Python、网页用 HTML/JS，三者天然冲突。
- 作者用 GitHub（共享源码）+ Jupyter（混合代码/公式/文本）+ Sphinx（多格式渲染）拼出一套集成工作流。
- 对学习者的启示：理论与代码必须绑定，不要只背公式或只抄代码。

### 2. 在实践中学习
- 「适时教学」：在你需要某个概念去实现目标的那一刻才引入它。
- 先体验训练模型的满足感，再深入更抽象的数学。
- 每节尽量是一个可独立运行的完整例子——复制即可改成自己的项目。
- 重要方法论：**从零实现 (仅张量 + 自动微分) + 框架简洁实现 (高级 API)** 两个版本对照。详见 [[概念-在实践中学习]]。

### 3. 内容和结构
- 全书分三部分：基础与预备 → 现代深度学习技术 → 可伸缩性/效率/应用。
- 完整结构与各章主题见 [[全书结构]]，个人学习顺序见 [[学习路径]]。

### 4. 代码
- 反复使用的函数/类封装进 `d2l` 包，用 `#@save` 标记保存。
- 本笔记体系**只关注 PyTorch 实现**。导入约定与机制详见 [[概念-d2l包]]。

### 5. 目标受众
- 面向学生、工程师、研究者；无需深度学习/机器学习背景。
- 前置只需：线性代数、微积分、概率、基础 Python。
- 风格上优先「直觉与想法」，其次才是数学严谨性。

## 易错点

- 把「从零实现」当成多余的体力活而跳过——其实它正是理解框架内部行为的关键。
- 只看简洁 API 版本，遇到 bug 时无法定位（不知道层/优化器内部发生了什么）。
- 误以为需要先学完所有数学才能开始——本书刻意反其道而行（先用后懂）。

## 和前后章节或真实项目的连接

- 前言 → 第 1 章「引言」：从动机过渡到深度学习的关键组成与发展史。
- 四要素中的「工程方法」直接对应后续 GPU、计算性能、混合精度等章节。
- 双版本方法论会贯穿全书：例如线性回归、softmax、MLP 都先从零再到简洁实现。
- 真实项目连接：理解框架内部行为是日后阅读/调试 FoundationPose、DINOv3 等真实仓库的基础。
