# 5.2-5.3 参数管理与（延后）初始化

## 一句话总结

**大白话**：模型里的权重就是一堆可调旋钮。本节学会怎么「点名查看/修改」这些旋钮、怎么给不同层设不同的初始值、怎么让两层共用同一组旋钮（参数共享），以及「先搭架子、等第一次数据进来才定输入维度」的延后初始化。

**严谨说法**：可通过 `state_dict` / `named_parameters` 访问参数，用 `nn.init` 自定义[[参数初始化|初始化]]，用同一层实例实现**参数共享**；`nn.LazyLinear` 支持延后初始化（首次前向时才确定输入维度）。

## 本节解决的问题

- 如何访问、遍历、修改某层参数？
- 如何对不同层施加不同初始化？
- 如何让多层共享同一组参数？

## 核心概念 / 公式 / API

| 操作 | API |
| --- | --- |
| 访问单层参数 | `net[2].weight.data`, `net[2].bias` |
| 遍历全部 | `net.named_parameters()` |
| 字典形式 | `net.state_dict()` |
| 自定义初始化 | `nn.init.normal_/constant_/xavier_uniform_` |
| 延后初始化 | `nn.LazyLinear(out)`（输入维度首次前向推断） |

### 初始化与共享
```python
import torch
from torch import nn

def init_xavier(m):
    if isinstance(m, nn.Linear):
        nn.init.xavier_uniform_(m.weight)
        nn.init.zeros_(m.bias)

shared = nn.Linear(8, 8)
net = nn.Sequential(nn.Linear(4, 8), nn.ReLU(),
                    shared, nn.ReLU(), shared,   # 参数共享
                    nn.Linear(8, 1))
net.apply(init_xavier)
```

## 易错点

- `param.data` 直接改值会绕过 autograd（初始化时可以，训练中谨慎）。
- 参数共享后，两处梯度会累加到同一份权重（这是预期行为）。
- LazyLinear 必须先跑一次前向才有实际参数，之前 `state_dict` 为空壳。

## 复习卡片

- Q: 如何遍历模型全部参数？
  A: `net.named_parameters()` 或 `net.state_dict()`。
- Q: 参数共享如何实现？
  A: 在 Sequential 中多次放入同一个层实例。
- Q: 延后初始化用什么层？
  A: `nn.LazyLinear`，首次前向才确定输入维度。
