# 13.1 图像增广 (Image Augmentation)

## 一句话总结

**大白话**：训练时把每张图随机翻转、裁剪、调色一下，相当于「一张图变出好多张」，让模型见多识广、不挑角度光照——本质是一种[[正则化]]，专治[[过拟合]]。注意测试时别随机（要可复现）。

**严谨说法**：图像增广在训练时随机对图像做翻转、裁剪、颜色抖动等变换，等效扩充数据、提升模型对位置/光照/尺度的鲁棒性，是降低[[过拟合]]的标准 CV 手段。

## 本节解决的问题

- 数据有限时如何提升泛化？
- 常见增广操作有哪些？
- 训练与测试的增广为何不同？

## 核心概念 / 公式 / API

| 增广 | API |
| --- | --- |
| 随机水平翻转 | `transforms.RandomHorizontalFlip()` |
| 随机裁剪缩放 | `transforms.RandomResizedCrop(size)` |
| 颜色抖动 | `transforms.ColorJitter(brightness, contrast, ...)` |
| 标准化 | `transforms.Normalize(mean, std)` |

- 训练用随机增广扩充多样性；测试通常只做确定性的 resize + center crop + normalize。
- 增广要符合任务语义（如数字识别不宜上下翻转）。

## 最小可运行代码

```python
from torchvision import transforms

train_aug = transforms.Compose([
    transforms.RandomResizedCrop(224),
    transforms.RandomHorizontalFlip(),
    transforms.ColorJitter(0.4, 0.4, 0.4),
    transforms.ToTensor(),
    transforms.Normalize([0.485, 0.456, 0.406], [0.229, 0.224, 0.225]),
])
test_aug = transforms.Compose([
    transforms.Resize(256), transforms.CenterCrop(224),
    transforms.ToTensor(),
    transforms.Normalize([0.485, 0.456, 0.406], [0.229, 0.224, 0.225]),
])
```

## 易错点

- 在测试集用随机增广 → 评估结果不稳定不可比。
- 增广破坏标签语义（如方向相关任务做翻转）。
- 忘记与预训练模型一致的 Normalize 统计量。

## 和前后章节 / 真实项目的连接

- 与微调配合用于小数据集 → [[微调]]。
- 正则化思想呼应 [[暂退法Dropout]]；自监督（DINOv3）大量依赖强增广构造视图。

## 复习卡片

- Q: 图像增广的核心收益？
  A: 等效扩充数据、提升鲁棒性、降低过拟合。
- Q: 训练与测试增广的差异？
  A: 训练随机增广，测试用确定性 resize+crop+normalize。
- Q: 选择增广要注意什么？
  A: 不能破坏任务的标签语义。
