# 10.5 多头注意力 (Multi-Head Attention)

## 一句话总结

**大白话**：与其用一双眼睛看，不如派好几组「眼睛」（头）同时看——每组关注不同的东西（有的盯语法、有的盯指代、有的盯远近），各看各的再把结论拼起来。这样一层就能同时捕捉多种关系，而总计算量和单头差不多。

**严谨说法**：多头[[注意力机制|注意力]]把 [[查询键值|query/key/value]] 线性投影到多个子空间并行做缩放点积注意力，让模型在不同表示子空间同时关注不同位置/特征，再拼接投影回原维度。

## 本节解决的问题

- 单头注意力的表达局限？
- 多头如何并行、如何拼接？
- 各头维度如何划分？

## 核心概念 / 公式 / API

- 把模型维度 $d$ 切成 h 个头，每头维度 $d/h$：

$$\mathrm{head}_i=\mathrm{Attention}(QW_i^Q, KW_i^K, VW_i^V)$$

$$\mathrm{MultiHead}=\mathrm{Concat}(\mathrm{head}_1,\dots,\mathrm{head}_h)W^O$$

- 每头在不同子空间学习不同关注模式（语法/指代/距离等）。
- 总计算量与单头（维度 d）相近，但表达更丰富。

## 输入输出形状

- 输入 `(B, n, d)` → 拆成 `(B, h, n, d/h)` 并行 → 拼回 `(B, n, d)`。

## 最小可运行代码

```python
import torch
from torch import nn

mha = nn.MultiheadAttention(embed_dim=128, num_heads=8, batch_first=True)
x = torch.randn(2, 10, 128)        # (B, seq, d)
out, attn_w = mha(x, x, x)         # 自注意力: Q=K=V=x
print(out.shape, attn_w.shape)     # (2,10,128) (2,10,10)
```

## 易错点

- `embed_dim` 必须能被 `num_heads` 整除。
- PyTorch `nn.MultiheadAttention` 默认 `batch_first=False`（输入 `(seq,B,d)`），设 `batch_first=True` 更直观。
- 误以为多头就是多套独立注意力——它们共享输入但投影到不同子空间。

## 和前后章节 / 真实项目的连接

- 内核是缩放点积 [[注意力评分函数]]。
- 自注意力 + 多头构成 Transformer 编码器层 → [[Transformer]]。

- 多头自注意力是 [[Transformer]] 与 [[BERT模型与预训练]] 的核心组件。

## 复习卡片

- Q: 多头注意力的核心动机？
  A: 在多个表示子空间并行关注不同模式。
- Q: 每个头的维度？
  A: 模型维度除以头数 $d/h$。
- Q: 各头输出如何合并？
  A: 拼接后再经一个线性投影 $W^O$。
