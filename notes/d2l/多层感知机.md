# 4.1-4.3 多层感知机与激活函数（从零 + 简洁）

## 一句话总结

**大白话**：把好几个线性层直接叠起来，其实还是一条直线（等于白叠）。在层与层之间塞一个「掰弯」的[[激活函数]]（如 ReLU），网络才能拟合各种弯弯曲曲的复杂函数。这就是从线性模型迈向「深度」的第一步。

**严谨说法**：多层感知机在线性层之间插入**非线性[[激活函数]]**（如 ReLU），从而拟合任意复杂函数；这是从线性模型迈向「深度」网络的第一步。

## 本节解决的问题

- 为什么多个线性层叠加仍是线性？激活函数解决了什么？
- 常见激活函数 ReLU/sigmoid/tanh 的特点？
- 隐藏层 MLP 的从零与简洁实现差异？

## 核心概念 / 公式 / API

### 为何需要非线性
- 两层线性 $W_2(W_1 x)=(W_2 W_1)x$ 仍是线性。必须插入非线性 $\sigma$：

$$\mathbf{h}=\sigma(\mathbf{W}_1\mathbf{x}+\mathbf{b}_1),\quad \mathbf{o}=\mathbf{W}_2\mathbf{h}+\mathbf{b}_2$$

| 激活 | 公式 | 特点 |
| --- | --- | --- |
| ReLU | $\max(0,x)$ | 计算快、缓解梯度消失，默认首选 |
| sigmoid | $1/(1+e^{-x})$ | 压缩到 (0,1)，易饱和梯度消失 |
| tanh | $(e^x-e^{-x})/(e^x+e^{-x})$ | 压缩到 (-1,1)，零中心 |

- 万能近似定理：单隐藏层 + 足够宽即可逼近任意连续函数（但深而窄通常更高效）。

## 输入输出形状

- `X(B,784) → 隐藏(B,256) → 输出(B,10)`。

## 最小可运行代码

仓库脚本：`d2l/code/4.1多层感知机.py`、`d2l/code/4.2多层感知机从零开始实现.py`、`d2l/code/4.3多层感知机的简洁实现.py`

从零（手写 ReLU）：
```python
import torch

def relu(X): return torch.max(X, torch.zeros_like(X))

W1 = torch.randn(784, 256, requires_grad=True) * 0.01
b1 = torch.zeros(256, requires_grad=True)
W2 = torch.randn(256, 10, requires_grad=True) * 0.01
b2 = torch.zeros(10, requires_grad=True)

def net(X):
    H = relu(X.reshape(-1, 784) @ W1 + b1)
    return H @ W2 + b2
```

简洁：
```python
from torch import nn

net = nn.Sequential(
    nn.Flatten(),
    nn.Linear(784, 256), nn.ReLU(),
    nn.Linear(256, 10),
)
```

## 易错点

- 隐藏层之间忘记加激活 → 整体退化为线性模型。
- 参数初始化全 0 → 对称性导致所有神经元学到相同特征。
- sigmoid/tanh 在深网络中易梯度消失，深层优先 ReLU。

## 复习卡片

- Q: 为什么 MLP 必须有非线性激活？
  A: 否则多层线性叠加仍等价单层线性。
- Q: 深层网络默认激活选哪个？为什么？
  A: ReLU，计算简单且缓解梯度消失。
- Q: 万能近似定理说明什么？
  A: 单隐藏层足够宽即可逼近任意连续函数。
