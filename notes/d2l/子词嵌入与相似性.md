# 14.4 / 14.6-14.7 子词嵌入、相似性与类比

## 一句话总结

**大白话**：遇到没见过的词怎么办？fastText 把词拆成「字符小片段」（子词），没见过的词也能用它的零件拼出向量。BPE 则用数据自动学一套「最划算的拆分方式」（现代分词主流）。学好的[[词嵌入|词向量]]还能玩「找近义词」和「类比」（国王-男人+女人≈王后），靠的是余弦相似度。

**严谨说法**：fastText 用**子词 (n-gram 字符)** 组合表示词，能处理未登录词与形态变化；BPE 用数据驱动的子词切分平衡[[词表]]大小与覆盖；训练好的词向量可用余弦相似度做近义词检索与「类比」推理。

## 本节解决的问题

- word2vec 遇到未登录词 (OOV) 怎么办？
- 子词如何捕获形态学信息？
- 如何用词向量做相似性/类比任务？

## 核心概念 / 公式 / API

### 子词嵌入
- **fastText**：词向量 = 其所有字符 n-gram 子词向量之和 → OOV 也能由子词拼出。
- **字节对编码 (BPE)**：从字符起反复合并最高频相邻对，得到子词词表（现代分词主流）。

### 相似性与类比
- 余弦相似度：$\cos(\mathbf{a},\mathbf{b})=\dfrac{\mathbf{a}^\top\mathbf{b}}{\lVert\mathbf{a}\rVert\lVert\mathbf{b}\rVert}$。
- 类比：`vec(man)-vec(woman) ≈ vec(king)-vec(queen)`，用向量加减 + 最近邻检索。

## 最小可运行代码

```python
import torch
import torch.nn.functional as F

def knn(W, query_vec, k):
    cos = F.cosine_similarity(W, query_vec[None, :], dim=1)
    return cos.topk(k)            # 返回最相似的 k 个词索引

# 类比 a:b :: c:?  ->  vec(c) + vec(b) - vec(a)
def analogy(vecs, a, b, c):
    return vecs[c] + vecs[b] - vecs[a]
```

## 易错点

- 不处理 OOV 直接映射到 `<unk>` 会丢信息，子词可缓解。
- 类比检索要排除查询词本身（否则返回自己）。
- 余弦相似度对向量是否归一化敏感。

## 复习卡片

- Q: fastText 如何处理未登录词？
  A: 用字符 n-gram 子词向量之和拼出词向量。
- Q: BPE 的核心操作？
  A: 反复合并最高频的相邻符号对形成子词。
- Q: 词类比如何用向量实现？
  A: 向量加减后做最近邻检索（余弦相似度）。
