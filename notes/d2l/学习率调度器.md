# 11.11 学习率调度器

## 一句话总结

**大白话**：[[学习率|步子大小]]不该从头到尾不变。常见套路是「先小步热身（warmup），再慢慢迈大、最后越走越小（衰减）」——开头别冲太猛免得摔，结尾小步微调免得在谷底反复横跳。这是大模型训练的关键调参手段。

**严谨说法**：学习率调度在训练过程中动态调整 η：常配合**预热 (warmup)** 起步 + 逐步衰减（多项式/余弦/分段），能显著提升收敛速度与最终精度，是大模型训练的关键工程手段。

## 本节解决的问题

- 为什么固定学习率不够好？
- 常见调度策略有哪些？
- 预热为什么对大模型/大 batch 重要？

## 核心概念 / 公式 / API

| 策略 | 说明 |
| --- | --- |
| 分段衰减 (StepLR) | 每隔若干 epoch 乘以 γ |
| 指数衰减 | $\eta_t=\eta_0\gamma^t$ |
| 余弦退火 | 按余弦曲线平滑降到接近 0 |
| 预热 (warmup) | 初期从小学习率线性升到目标值，稳住早期训练 |

- 预热避免训练初期大梯度+大学习率导致发散，尤其大 batch / Transformer。
- 典型组合：warmup + 余弦衰减。

| API | 说明 |
| --- | --- |
| `torch.optim.lr_scheduler.StepLR` | 分段 |
| `CosineAnnealingLR` | 余弦退火 |
| `LambdaLR` | 自定义（可实现 warmup） |

## 最小可运行代码

```python
import torch

opt = torch.optim.SGD(net.parameters(), lr=0.1)
sched = torch.optim.lr_scheduler.CosineAnnealingLR(opt, T_max=50)

for epoch in range(50):
    # train_one_epoch(...)
    opt.step()
    sched.step()                # 每个 epoch 更新学习率
    print(sched.get_last_lr())
```

## 易错点

- `scheduler.step()` 调用时机/频率错误（每 epoch vs 每 step）导致衰减过快或不变。
- 新版 PyTorch 要求先 `optimizer.step()` 再 `scheduler.step()`。
- warmup 步数过短对大模型仍可能早期发散。

## 复习卡片

- Q: warmup 的作用？
  A: 初期用小学习率稳住训练，避免早期发散。
- Q: 常见的衰减曲线有哪些？
  A: 分段、指数、余弦退火。
- Q: PyTorch 中 step 的正确顺序？
  A: 先 `optimizer.step()` 再 `scheduler.step()`。
