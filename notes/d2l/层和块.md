# 5.1 层和块 (Layers and Blocks)

## 一句话总结

**大白话**：在 PyTorch 里，不管是一个小零件（层）、一组零件（块）、还是整台机器（网络），都用同一种「积木」`nn.Module` 表示。想自己造积木，只要继承它、在 `__init__` 里登记有哪些子零件、在 `forward` 里写「数据怎么流过去」就行。

**严谨说法**：`nn.Module` 是 PyTorch 组织模型的统一抽象：单层、由层组成的块、整个网络都是 Module；自定义块只需继承它并实现 `__init__`（注册子模块）和 `forward`（定义前向）。

## 本节解决的问题

- `nn.Sequential` 内部如何工作？
- 如何自定义带控制流的前向计算？
- 块如何嵌套组合成复杂网络？

## 核心概念 / 公式 / API

| 概念 | 说明 |
| --- | --- |
| `nn.Module` | 一切模型/层的基类，自动追踪子模块与参数 |
| `forward` | 定义前向计算（不要直接调用，用 `net(X)`） |
| `nn.Sequential` | 按顺序串联子模块的容器 |
| 嵌套 | Module 内可放 Module，递归组合 |

- 把子模块赋值给 `self.xxx` 会自动注册其参数到父模块。

## 最小可运行代码

```python
import torch
from torch import nn
from torch.nn import functional as F

class MLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.hidden = nn.Linear(20, 256)   # 子模块自动注册
        self.out = nn.Linear(256, 10)
    def forward(self, X):
        return self.out(F.relu(self.hidden(X)))

class FixedHidden(nn.Module):
    def __init__(self):
        super().__init__()
        self.rand_w = torch.rand(20, 20)   # 不更新的常量
        self.linear = nn.Linear(20, 20)
    def forward(self, X):
        X = F.relu(X @ self.rand_w + 1)    # 自定义控制流
        return self.linear(X).sum()

net = MLP()
print(net(torch.rand(2, 20)).shape)        # (2, 10)
```

## 易错点

- 忘记 `super().__init__()` → 参数无法注册。
- 直接调用 `net.forward(X)` 而非 `net(X)`，绕过钩子机制。
- 把可学习参数写成普通张量（未用 `nn.Parameter`/子模块）→ 不被优化器追踪。

## 复习卡片

- Q: 自定义块要实现哪两个方法？
  A: `__init__`（注册子模块）和 `forward`（前向）。
- Q: 为什么用 `net(X)` 而不是 `net.forward(X)`？
  A: `__call__` 会触发注册的钩子等机制。
- Q: 子模块参数如何被追踪？
  A: 赋值给 `self.xxx` 后自动注册到父 Module。
