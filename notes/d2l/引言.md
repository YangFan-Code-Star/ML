# 第1章 引言 (Introduction)

## 一句话总结

**大白话**：传统程序是人把规则一条条写死；机器学习是给它一堆例子，让它自己总结规则。要素就四样：例子（数据）、一个带旋钮的函数（模型）、一把打分尺（目标函数）、一套拧旋钮的方法（优化）。深度学习就是把「函数」叠很多层，让它自己学会从原始数据里提取有用特征。

**严谨说法**：机器学习是用**数据 + 模型 + 目标函数 + 优化算法**自动从经验中学习规律的范式；深度学习是其中用「多层表示学习」自动提取特征的子集，本章建立全书的术语与全局认知。

## 本节解决的问题

- 什么是机器学习？它和「写死规则的程序」有何本质区别？
- 一个机器学习问题由哪些关键组件构成？
- 有哪些学习范式（监督 / 无监督 / 强化）？各自解决什么问题？
- 深度学习为什么近年才爆发（数据、算力、算法三者合力）？

## 核心概念 / 公式 / API

### 机器学习四个关键组件
| 组件               | 含义                            | 后续对应                               |
| ---------------- | ----------------------------- | ---------------------------------- |
| 数据 (data)        | 样本 + 特征/标签；通常假设独立同分布（[[IID假设\|i.i.d.]]） | [[数据操作与预处理]] |
| 模型 (model)       | 把输入映射到预测的参数化函数                | 线性回归 → MLP → CNN/RNN               |
| 目标函数 (objective) | 衡量预测好坏的损失，越小越好                | [[线性回归]]      |
| 优化算法 (algorithm) | 调整参数最小化损失，核心是 SGD             | [[梯度下降与SGD]]   |

### 监督学习 (supervised)
- 给定「特征 → 标签」对，学习映射。
- 子类型：**回归**（连续值，如房价）、**分类**（离散类别，如图像分类）、标注、搜索、推荐、序列学习（机器翻译）。

### 无监督学习 (unsupervised)
- 只有数据没有标签：聚类、主成分分析、生成模型 (GAN/VAE)、自监督表示学习。
- 对应真实项目：DINOv3 等自监督视觉表示学习。

### 强化学习 (reinforcement)
> **大白话**：像训小狗——没有标准答案，只有「做对给糖、做错挨罚」，靠反复试错学会怎么做最划算。

- 智能体 (agent) 与环境交互，通过动作获得奖励，学习策略最大化累积回报。详见 [[强化学习]]。

### 深度学习的本质
- 用**多层非线性变换**自动学习从原始数据到任务输出的**表示 (representation)**，取代人工特征工程。
- 端到端 (end-to-end) 训练：从原始像素/文本直接学到目标。

## 易错点

- 把「监督学习 = 分类」搞混：回归也是监督学习。
- 误以为深度学习总优于传统方法——小数据/强先验场景下未必。
- 忽视 i.i.d. 假设；现实中分布偏移会破坏它 → [[分布偏移]]。

## 复习卡片

- Q: 机器学习的四个关键组件？
  A: 数据、模型、目标函数、优化算法。
- Q: 回归与分类的区别？
  A: 回归预测连续值，分类预测离散类别，二者都是监督学习。
- Q: 深度学习相对传统 ML 的核心优势？
  A: 自动多层表示学习，免去人工特征工程，可端到端训练。
