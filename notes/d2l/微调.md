# 13.2 微调 (Fine-Tuning)

## 一句话总结

**大白话**：别从零养孩子，直接请个「在海量数据上见过世面」的预训练模型来——它底层认边缘、纹理的本事是通用的，可以直接借用。只把最后那层「下结论」的换成你的任务，再用小[[学习率]]轻轻调一调，小数据集也能又快又好。这就是[[迁移学习]]。

**严谨说法**：微调是[[迁移学习]]的核心做法：复用在大数据集（如 ImageNet）预训练的骨干特征，只替换输出层并以较小学习率继续训练，能在小数据集上又快又好，是工业落地的首选范式。

## 本节解决的问题

- 小数据集从头训练为何效果差？
- 如何复用预训练模型的特征？
- 输出层与骨干的学习率为何要分开设？

## 核心概念 / 公式 / API

### 微调步骤
1. 取预训练模型（保留骨干权重）。
2. 替换输出层为目标任务类别数（随机初始化）。
3. 用**小学习率**微调骨干、**较大学习率**训练新输出层。
4. 输入需用与预训练一致的归一化。

- 直觉：底层特征（边缘/纹理）通用，可迁移；高层与输出层任务相关，需重学。

## 最小可运行代码

```python
import torch
from torch import nn
from torchvision import models

net = models.resnet18(weights=models.ResNet18_Weights.DEFAULT)
net.fc = nn.Linear(net.fc.in_features, 2)     # 替换输出层为 2 类
nn.init.xavier_uniform_(net.fc.weight)

# 输出层用更大学习率，骨干用小学习率
opt = torch.optim.SGD([
    {"params": [p for n, p in net.named_parameters() if "fc" not in n], "lr": 1e-4},
    {"params": net.fc.parameters(), "lr": 1e-3},
], momentum=0.9)
```

## 易错点

- 新输出层与骨干用同一大学习率 → 破坏预训练特征。
- 输入归一化与预训练不一致 → 特征分布错位。
- 数据极少时甚至应冻结骨干只训输出层。

## 复习卡片

- Q: 微调为什么对小数据集有效？
  A: 复用预训练通用底层特征，只需重学任务相关部分。
- Q: 骨干与新输出层学习率怎么设？
  A: 骨干小、输出层大。
- Q: 微调时输入要注意什么？
  A: 用与预训练一致的归一化。
