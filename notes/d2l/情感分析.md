# 15.1-15.3 情感分析（RNN 与 CNN）

## 一句话总结

**大白话**：判断一段评论是「好评还是差评」，本质就是文本分类。两条路：双向 RNN 把整句读完取「读后感」（末隐状态）来分类；或 textCNN 用滑窗抓「局部短语特征」（n-gram）再取最显著的来分类。两者都拿预训练[[词嵌入|词向量]]当输入。

**严谨说法**：情感分析是文本分类任务；可用双向 RNN 编码整句取末[[隐状态]]分类，或用一维[[卷积]] (textCNN) 提取局部 n-gram 特征再最大池化分类，二者都以预训练[[词嵌入]]作输入。

## 本节解决的问题

- 文本分类的标准流程（嵌入→编码→分类头）？
- RNN 与 CNN 处理文本各自的特点？
- textCNN 的一维卷积/最大池化如何工作？

## 核心概念 / 公式 / API

| 方案 | 编码方式 | 特点 |
| --- | --- | --- |
| BiRNN | 双向 LSTM 取首尾隐状态拼接 | 捕获长程顺序依赖 |
| textCNN | 多种宽度一维卷积 + 时序最大池化 | 抓局部 n-gram，快且并行 |

- 输入：预训练词向量（如 GloVe）经 `nn.Embedding` 加载，可冻结或微调。
- textCNN：不同 kernel 宽度 = 不同 n-gram 窗口，池化后拼接送全连接。

## 输入输出形状

- 词索引 `(B, L)` → 嵌入 `(B, L, d)` → 编码 → 分类 logits `(B, 2)`。

## 最小可运行代码

```python
import torch
from torch import nn

class TextCNN(nn.Module):
    def __init__(self, vocab, embed, kernels, channels, num_classes=2):
        super().__init__()
        self.embedding = nn.Embedding(vocab, embed)
        self.convs = nn.ModuleList([
            nn.Conv1d(embed, c, k) for c, k in zip(channels, kernels)])
        self.pool = nn.AdaptiveMaxPool1d(1)
        self.fc = nn.Linear(sum(channels), num_classes)
    def forward(self, x):
        emb = self.embedding(x).permute(0, 2, 1)   # (B, embed, L)
        feats = [self.pool(torch.relu(conv(emb))).squeeze(-1) for conv in self.convs]
        return self.fc(torch.cat(feats, dim=1))
```

## 易错点

- 一维卷积要求把嵌入维度放到通道维 → 需 `permute`。
- 变长文本需 padding + 池化处理，注意掩码。
- 预训练嵌入与当前词表对齐（未命中词随机初始化）。

## 复习卡片

- Q: 情感分析属于哪类任务？
  A: 文本（序列）分类。
- Q: textCNN 不同 kernel 宽度对应什么？
  A: 不同长度的 n-gram 局部特征。
- Q: 一维卷积输入为何要 permute？
  A: 把嵌入维放到通道维 `(B, embed, L)`。
