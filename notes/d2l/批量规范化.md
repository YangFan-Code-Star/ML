# 7.5 批量规范化 (Batch Normalization)

## 一句话总结

**大白话**：深网络里，每一层的输入分布会随训练不停乱晃，后面层很难适应。批量规范化就是在每一层前「把数据拉回标准状态」（减均值、除标准差），再给两个可学习旋钮 γ、β 让网络按需缩放。好处：训得更快、能用更大学习率、还顺带有点[[正则化]]效果。

**严谨说法**：批量规范化对每个小批量的中间激活做标准化（零均值单位方差）再用可学习的 γ、β 缩放平移，稳定每层输入分布，从而**加速收敛、允许更大学习率、并有轻微正则效果**。

## 本节解决的问题

- 深层网络中间层分布漂移导致难训练，怎么办？
- BN 在训练与推理时行为为何不同？
- 全连接层与卷积层的 BN 维度差异？

## 核心概念 / 公式 / API

### BN 公式
对小批量 $\mathcal{B}$：

$$\hat{x}=\frac{x-\mu_{\mathcal{B}}}{\sqrt{\sigma_{\mathcal{B}}^2+\epsilon}},\quad y=\gamma\hat{x}+\beta$$

- γ、β 可学习；推理时用训练阶段累计的**移动平均**均值/方差。

| 场景 | 规范化维度 |
| --- | --- |
| 全连接 | 对每个特征维（dim=0 样本方向） |
| 卷积 | 对每个通道（在 N,H,W 上统计） |

| API | 说明 |
| --- | --- |
| `nn.BatchNorm1d(num_features)` | 全连接后 |
| `nn.BatchNorm2d(num_channels)` | 卷积后 |

- 训练 `net.train()` 用当前 batch 统计；推理 `net.eval()` 用移动平均。

## 最小可运行代码

```python
from torch import nn

net = nn.Sequential(
    nn.Conv2d(1, 6, 5), nn.BatchNorm2d(6), nn.ReLU(), nn.MaxPool2d(2, 2),
    nn.Conv2d(6, 16, 5), nn.BatchNorm2d(16), nn.ReLU(), nn.MaxPool2d(2, 2),
    nn.Flatten(),
    nn.Linear(16 * 4 * 4, 120), nn.BatchNorm1d(120), nn.ReLU(),
    nn.Linear(120, 10),
)
```

## 易错点

- 推理忘记 `eval()` → 用了 batch 统计，小 batch 时结果抖动。
- batch 太小时 BN 估计不稳；可考虑 LayerNorm/GroupNorm。
- BN 与 dropout 叠加顺序需注意，通常 BN 在激活前。

## 复习卡片

- Q: BN 的主要收益？
  A: 稳定分布、加速收敛、允许更大学习率、轻微正则。
- Q: 推理时 BN 用什么统计量？
  A: 训练期累计的移动平均均值/方差。
- Q: 卷积层 BN 在哪些维度统计？
  A: 每个通道，在 N、H、W 上统计。
