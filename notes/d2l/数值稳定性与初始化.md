# 4.7-4.8 前向/反向传播、数值稳定性与模型初始化

## 一句话总结

**大白话**：深网络求[[梯度]]时要把很多层的数一路连乘——都小于 1 越乘越小（梯度消失，前面层学不动），都大于 1 越乘越大（梯度爆炸，数值炸了）。解决靠：选不易饱和的激活（ReLU）+ 把初始权重的大小调得「不大不小」（Xavier/He 初始化），让每层的方差别失控。

**严谨说法**：反向传播按计算图链式求导；深网络中梯度连乘易**消失或爆炸**，合理的激活选择与**参数初始化（Xavier）**可让各层方差稳定，是能否训得动的关键。

## 本节解决的问题

## 核心概念 / 公式 / API

### 前向 / 反向传播
- 前向：按计算图从输入算到损失，缓存中间结果。
- 反向：从损失沿图反向，用链式法则累乘局部梯度。
- 训练显存主要被前向缓存的中间激活占用。

### 梯度消失 / 爆炸
> **大白话**：想象一条很长的传话链，每个人都把听到的音量乘个系数再传。系数都小于 1，传到头就听不见了（消失）；都大于 1，传到头就震耳欲聋（爆炸）。深网络的梯度就是这么一层层连乘出来的。

- 深层梯度是多个雅可比矩阵连乘（见 [[梯度]]）：
  - 连乘 <1 → 梯度**消失**（sigmoid 饱和区导数→0）。
  - 连乘 >1 → 梯度**爆炸**（数值溢出）。

### 参数初始化
- 目标：让每层输出与梯度的方差跨层大致不变。
- **Xavier/Glorot**：$\mathrm{Var}(w)=\dfrac{2}{n_{in}+n_{out}}$，适配 tanh。
- **Kaiming/He**：适配 ReLU，方差 $2/n_{in}$。

```python
import torch.nn as nn
def init_weights(m):
    if isinstance(m, nn.Linear):
        nn.init.xavier_uniform_(m.weight)
        nn.init.zeros_(m.bias)
net.apply(init_weights)
```

## 易错点

- 全 0 或全相同初始化 → 对称性破坏不了，神经元学到相同特征。
- 初始化方差过大 → 激活/梯度爆炸；过小 → 消失。
- 深层用 sigmoid 极易梯度消失，优先 ReLU + Kaiming 初始化。

## 复习卡片

- Q: 梯度消失的常见根源？
  A: sigmoid 饱和区导数趋 0，深层连乘后梯度趋 0。
- Q: Xavier 初始化的目标？
  A: 让前向输出与反向梯度的方差跨层保持稳定。
- Q: 为什么不能零初始化权重？
  A: 对称性导致所有神经元更新一致，无法学到不同特征。
