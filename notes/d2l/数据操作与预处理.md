# 2.1-2.2 数据操作与预处理

## 一句话总结

**大白话**：张量就是「能装很多个数、还能整块一起算」的多维数组——一个数是 0 维、一行数是 1 维、一张表是 2 维、一摞表是 3 维。先学会怎么造它、切它、算它，再学会把脏数据（带缺失值的 CSV）洗成张量，后面一切才能开工。

**严谨说法**：张量 (`torch.Tensor`) 是深度学习的基本数据结构；掌握创建、索引、运算、[[广播]]与就地操作，以及用 `pandas` 把原始 CSV 清洗成可输入[[张量]]，是后续一切的基础。

## 本节解决的问题

- 如何在 PyTorch 中创建、变形、索引、运算张量？
- 广播 (broadcasting) 机制如何让不同形状的张量相加？
- 如何避免不必要的内存分配（就地操作）？
- 如何把含缺失值的原始数据预处理成张量？

## 核心概念 / 公式 / API

| 操作 | API | 说明 |
| --- | --- | --- |
| 创建 | `torch.arange / zeros / ones / randn / tensor` | 指定形状或从列表创建 |
| 形状 | `x.shape`, `x.numel()`, `x.reshape(-1, 4)` | `-1` 自动推断维度 |
| 运算 | `+ - * / **`, `torch.exp`, `torch.cat`, `==` | 逐元素；`cat(dim=)` 拼接 |
| 降维 | `x.sum()`, `x.mean(dim=0)` | 沿轴归约 |
| 索引 | `x[-1]`, `x[1:3]`, `x[:, 1] = 9` | 与 NumPy 一致 |
| 互转 | `x.numpy()`, `torch.from_numpy(a)`, `x.item()` | 张量↔NumPy↔标量 |

### 广播机制
> **大白话**：两个形状不一样的张量想逐元素相加，系统会把「只有 1 行/1 列」的那个自动复制铺开，凑成一样大再算。省得你手动复制。

- 两个张量从**尾部维度**对齐，维度为 1 或缺失的可被「复制扩展」后逐元素运算。
- 例：`(3,1) + (1,2) → (3,2)`。详见 [[广播]]。

### 就地操作（省内存）
```python
Z = torch.zeros_like(Y)
Z[:] = X + Y      # 写入预分配内存
X += Y            # 等价就地，避免新分配
```

## 输入输出形状

- `reshape`：元素总数不变；`(12,) → (3,4)`。
- 广播：尾维对齐后扩展，输出取各维最大值。

## 最小可运行代码

仓库脚本：`d2l/code/2.1数据操作.py`、`d2l/code/2.2数据预处理.py`

```python
import os
import pandas as pd
import torch

x = torch.arange(12, dtype=torch.float32).reshape(3, 4)
print(x.sum(), x.mean(dim=0))           # 归约
print(torch.cat([x, x], dim=1).shape)   # 拼接 -> (3, 8)

# 预处理：读取 CSV，填补缺失值，转为张量
data = pd.read_csv("house.csv")              # 含 NaN
inputs = data.iloc[:, :-1].fillna(data.iloc[:, :-1].mean(numeric_only=True))
inputs = pd.get_dummies(inputs, dummy_na=True)  # 类别变量独热
X = torch.tensor(inputs.to_numpy(dtype=float))
```

## 易错点

- `reshape` 与原张量**共享内存**，修改会相互影响。
- 广播会「悄悄」扩展形状导致非预期结果，调试时先打印 `shape`。
- `x + y` 会分配新内存；循环里反复加法应用就地 `+=`。

## 复习卡片

- Q: 广播的对齐规则？
  A: 从尾部维度对齐，维度为 1 或缺失者扩展复制。
- Q: 如何省内存做逐元素加？
  A: 用 `X[:] = X + Y` 或 `X += Y` 就地写入。
- Q: 处理类别型缺失值常用哪个函数？
  A: `pd.get_dummies(..., dummy_na=True)`。
