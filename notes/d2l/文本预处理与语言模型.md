# 8.2-8.3 文本预处理与语言模型

## 一句话总结

**大白话**：模型只认数字，不认字。所以要先把文本「切成一个个词/字」（分词）、给每个词编号（建[[词表]]）、再把整段话变成一串编号。语言模型干的事是「猜下一个词」——n 元语法靠数频率（只看前几个词），但词频是长尾的、组合一多就没几条样本可数（稀疏），所以最终还得靠神经网络。

**严谨说法**：文本要先**分词 → 建[[词表]]（token↔索引）→ 数值化**才能输入网络；语言模型估计序列概率，n 元语法用马尔可夫假设近似，但受稀疏性限制，引出神经网络语言模型。

## 本节解决的问题

- 原始文本如何变成模型可读的索引序列？
- 词表如何处理低频词与特殊符号（unk/pad/bos/eos）？
- n 元语法的优缺点与齐普夫定律？

## 核心概念 / 公式 / API

### 预处理流程
1. 读取文本，按字符或单词**分词 (tokenize)**。
2. 统计词频构建**词表 (Vocab)**：token → 整数索引，低频词归 `<unk>`。
3. 把语料映射为索引序列；批量训练时做小批量采样（随机/顺序分区）。

### 语言模型
- 目标：$P(x_1,\dots,x_T)=\prod_t P(x_t\mid x_{<t})$。
- **n 元语法**：用前 n-1 个词近似，统计频率估计（拉普拉斯平滑）。
- 词频服从**齐普夫定律**（长尾），高阶 n-gram 严重稀疏 → 需要神经网络。

## 最小可运行代码

```python
import collections

def tokenize(lines): return [list(line) for line in lines]   # 字符级

class Vocab:
    def __init__(self, tokens, min_freq=0):
        counter = collections.Counter([t for line in tokens for t in line])
        self.itos = ["<unk>"] + [t for t, c in counter.items() if c >= min_freq]
        self.stoi = {t: i for i, t in enumerate(self.itos)}
    def __getitem__(self, t):
        return self.stoi.get(t, 0)     # 未知词 -> <unk>=0
```

## 易错点

- 测试集出现训练词表外的词，未用 `<unk>` 处理会越界。
- 批量采样若跨样本边界拼接，注意区分随机分区与顺序分区。
- 字符级 vs 单词级词表粒度影响序列长度与词表大小。

## 复习卡片

- Q: 文本数值化三步？
  A: 分词 → 建词表 → 映射为索引。
- Q: n 元语法的主要缺陷？
  A: 高阶时数据稀疏（齐普夫长尾），难估计。
- Q: `<unk>` 的作用？
  A: 表示词表外的未知 token。
