# 查询/键/值 (Query / Key / Value)

> **大白话**：像查字典。**查询(query)** = 你要找的问题；**键(key)** = 每个词条的索引标签；**值(value)** = 词条的实际内容。拿 query 和每个 key 比相似度定权重，再用权重去加权对应的 value。

## 定义
注意力的三要素：
- **Query**：当前要计算输出的位置发出的「问题」向量；
- **Key**：每个被检索项的「索引」向量，用于和 query 算相似度；
- **Value**：每个被检索项的「内容」向量，是真正被加权汇总的对象。

注意：key 用于打分，value 才是被聚合的内容，二者可相同也可不同。

## 不同场景的 QKV 来源
| 场景 | Q | K=V |
| --- | --- | --- |
| Bahdanau 翻译 | 解码器隐状态 | 编码器各步输出 |
| 自注意力 | 序列自身 | 序列自身 |
| 编码器-解码器注意力 | 解码器 | 编码器输出 |
