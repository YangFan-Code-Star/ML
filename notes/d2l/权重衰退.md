# 4.5 权重衰退 (Weight Decay / L2 正则)

## 一句话总结

**大白话**：在损失里额外加一条「别把权重搞太大」的罚款（[[正则化]]的一种），权重越大罚得越狠。效果是每次更新前先把权重「缩水」一点点，逼模型变简单，从而少[[过拟合]]。

**严谨说法**：权重衰退在损失中加入 $\frac{\lambda}{2}\lVert\mathbf{w}\rVert^2$ 的 L2 惩罚（见 [[范数]]），限制权重幅度以降低模型复杂度，缓解过拟合；在 SGD 中等价于每步对权重乘以一个略小于 1 的衰减因子。

## 本节解决的问题

- 如何用范数惩罚控制模型复杂度？
- L2 正则为何叫「权重衰退」？
- 在 PyTorch 优化器里如何一行开启？

## 核心概念 / 公式 / API

### 带正则的损失

$$L(\mathbf{w},b)+\frac{\lambda}{2}\lVert\mathbf{w}\rVert_2^2$$

- 梯度更新：

$$\mathbf{w}\leftarrow(1-\eta\lambda)\mathbf{w}-\eta\nabla_{\mathbf{w}}L$$

即每步先把权重「衰退」到 $(1-\eta\lambda)$ 倍，故名权重衰退。

- $\lambda$ 越大惩罚越强，权重越接近 0，模型越简单。通常只对权重不对偏置 `b` 正则。

### 简洁实现
- `torch.optim.SGD(params, lr=..., weight_decay=λ)` 直接内置。
- 想只对权重正则，用参数分组：
```python
trainer = torch.optim.SGD([
    {"params": net[0].weight, "weight_decay": wd},
    {"params": net[0].bias},
], lr=lr)
```

## 最小可运行代码

仓库脚本：`d2l/code/4.5权重衰退.py`

从零（手动加 L2 惩罚）：
```python
def l2_penalty(w):
    return (w ** 2).sum() / 2

# 训练步内
l = loss(net(X), y) + lambd * l2_penalty(w)
l.backward()
```

简洁（优化器内置）：
```python
import torch
from torch import nn
net = nn.Sequential(nn.Linear(200, 1))
trainer = torch.optim.SGD(net.parameters(), lr=0.003, weight_decay=3)
```

## 易错点

- 对偏置也施加正则通常无益，建议参数分组只衰退权重。
- `weight_decay` 太大导致欠拟合（权重被压得过小）。
- 注意 L2 正则系数与学习率耦合，调参时一起考虑。

## 和前后章节 / 真实项目的连接

- 与 [[暂退法Dropout]] 并列为两大正则手段，作用机制不同。
- L2 范数定义见 [[线性代数]]；现代优化器 AdamW 把权重衰退与梯度解耦 → [[Adam]]。

- 另一种常用正则化手段：[[暂退法Dropout]]（训练时随机置零激活）。

## 复习卡片

- Q: 权重衰退在更新公式里表现为什么？
  A: 每步权重先乘 $(1-\eta\lambda)$ 再减梯度。
- Q: 为什么一般不对偏置正则？
  A: 偏置不影响复杂度，正则收益小。
- Q: PyTorch 如何开启？
  A: 优化器的 `weight_decay` 参数。
