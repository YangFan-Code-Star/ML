# 11.3-11.5 梯度下降、随机梯度下降与小批量

## 一句话总结

**大白话**：下山每一步朝「最陡下坡方向」（负[[梯度]]）迈。算这个方向时：用全部数据最准但每步太慢；只用一条数据（SGD）快但方向抖；折中用一小撮数据（小批量 SGD），又快又稳还能并行——这是深度学习的默认做法。步子大小（学习率）是最关键的旋钮。

**严谨说法**：梯度下降沿负[[梯度]]更新参数；用全量数据太慢，用单样本 (SGD) 噪声大，**小批量 SGD** 兼顾计算效率（向量化/并行）与梯度估计稳定，是深度学习的默认优化方式。

## 本节解决的问题

- 学习率如何影响收敛（太大发散、太小慢）？
- GD / SGD / 小批量 SGD 的取舍？
- 批量大小对训练的影响？

## 核心概念 / 公式 / API

### 更新规则

$$\mathbf{w}\leftarrow\mathbf{w}-\eta\nabla_{\mathbf{w}}L$$

| 方法 | 每步用多少样本 | 特点 |
| --- | --- | --- |
| 批量 GD | 全部 | 梯度准但每步慢 |
| SGD | 1 | 快但噪声大、难向量化 |
| 小批量 SGD | B（如 32-256） | 兼顾速度与稳定，可并行 |

- 学习率 η 是最关键超参；过大震荡/发散，过小收敛慢。
- 小批量随机性还有轻微正则、帮助逃离鞍点。

## 最小可运行代码

```python
import torch

# 小批量 SGD（手写）
def sgd(params, lr, batch_size):
    with torch.no_grad():
        for p in params:
            p -= lr * p.grad / batch_size
            p.grad.zero_()

# 框架版
opt = torch.optim.SGD(net.parameters(), lr=0.05)
```

## 易错点

- 不同 batch size 下最优学习率不同，调参需联动。
- SGD 梯度未按 batch_size 归一时学习率尺度会偏。
- 学习率固定不变常导致后期震荡，需调度（见 [[学习率调度器]]）。

## 复习卡片

- Q: 小批量 SGD 为什么是默认选择？
  A: 兼顾计算效率（向量化）与梯度估计稳定性。
- Q: 学习率过大/过小各有什么问题？
  A: 过大震荡发散，过小收敛缓慢。
- Q: 小批量随机性的额外好处？
  A: 轻微正则、帮助逃离鞍点。
