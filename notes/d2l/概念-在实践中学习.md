# 在实践中学习 & 双版本实现

> 返回 [[学习地图MOC]] | 来源 [[前言精读]]

## 一句话总结

**大白话**：用到啥才学啥（别一上来背全套理论），而且每个核心零件都「先自己手搓、再用框架一行搞定」——手搓让你看清里头在干嘛，框架让你写得快。

**严谨说法**：本书采用「适时教学」——在你需要某概念去实现目标的那一刻才引入它；并对核心组件提供**从零实现**与**框架简洁实现**两个版本，先理解内部机制再使用高级 API。

## 方法论一：适时教学 (just-in-time)

- 传统教科书（如 Bishop 的书）每个主题都极其透彻，但初学者读到「线性回归」就要翻越大量铺垫。
- 本书反过来：先花少量篇幅讲线性代数/概率等背景，然后**尽快让你体验训练模型的满足感**。
- 每节尽量是一个「可独立运行的完整例子」，使用真实数据集，复制即可改成自己的项目。

## 方法论二：从零实现 vs 简洁 API 双版本

> 这是贯穿全书最重要的学习策略。

| 维度 | 从零实现 (from scratch) | 简洁实现 (concise) |
| --- | --- | --- |
| 依赖 | 仅张量操作 + 自动微分 | 框架高级 API (`nn.Module`, 优化器, `DataLoader`) |
| 目的 | 看清层 / 优化器内部发生了什么 | 贴近工程实践，代码简洁 |
| 适用阶段 | 第一次学某组件时 | 理解原理之后的后续章节 |
| 风险 | 代码长、易写错 | 黑箱，出 bug 难定位 |

### 示例：同一个线性层的两种写法（PyTorch）

从零实现（手动管理参数与前向）：

```python
import torch

# 参数：手动创建并开启梯度
w = torch.normal(0, 0.01, size=(num_inputs, 1), requires_grad=True)
b = torch.zeros(1, requires_grad=True)

def linreg(X):
    # 手写前向：y = X @ w + b
    return X @ w + b
```

简洁实现（用高级 API）：

```python
from torch import nn

# 一行搞定线性层，参数与前向由框架管理
net = nn.Sequential(nn.Linear(num_inputs, 1))
net[0].weight.data.normal_(0, 0.01)
net[0].bias.data.fill_(0)
```

- 两者数学上等价，但从零版让你明白 `nn.Linear` 内部就是 `X @ w + b`。
- 框架导入与 `d2l` 包约定见 [[概念-d2l包]]。

## 与四要素的关系

- 「从零实现」强训**数学**（要素 2）与**优化算法**（要素 3）的理解。
- 「简洁实现」强训**工程方法**（要素 4）。
- 四要素详见 [[概念-四要素]]。

## 易错点

- 跳过从零实现，导致只会调库、不懂原理。
- 在从零实现里忘记 `requires_grad=True` 或忘记 `optimizer.zero_grad()`，梯度行为异常。

## 复习卡片

- Q: 「适时教学」是什么意思？
  A: 在需要某概念去实现目标的那一刻才引入它，先实践后理论。
- Q: 双版本实现各自的价值？
  A: 从零版揭示内部机制，简洁版贴近工程；先懂再用。
- Q: `nn.Linear(n,1)` 内部数学等价于什么？
  A: `X @ w + b`。
