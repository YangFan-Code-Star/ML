# 4.4 模型选择、欠拟合与过拟合

## 一句话总结

**大白话**：模型太「笨」（容量低）连训练题都做不好叫欠拟合；太「死记硬背」（容量高）把训练题背得滚瓜烂熟、遇到新题就抓瞎叫[[过拟合]]。判断标准是看「训练分」和「新题分」的差距。挑模型/超参要用单独的验证集，数据少就用 K 折轮着验证。

**严谨说法**：模型容量与数据量决定泛化：容量过低→欠拟合，过高→[[过拟合]]；用**验证集 / K 折交叉验证**做模型选择，关注训练误差与[[泛化误差]]的差距。

## 本节解决的问题

- 训练误差与泛化误差的区别？
- 如何判断欠拟合 vs 过拟合？
- 没有大量数据时如何可靠地选超参数（K 折）？

## 核心概念 / 公式 / API

| 概念 | 含义 |
| --- | --- |
| 训练误差 | 模型在训练集上的误差 |
| 泛化误差 | 在新数据（同分布）上的期望误差 |
| 验证集 | 用于调超参，不能用测试集调参 |
| K 折交叉验证 | 数据分 K 份轮流验证，平均结果，适合小数据 |

### 欠拟合 vs 过拟合
| 现象 | 训练误差 | 泛化差距 | 对策 |
| --- | --- | --- | --- |
| 欠拟合 | 高 | 小 | 增大容量/特征/训练更久 |
| 过拟合 | 低 | 大 | 正则化/更多数据/降容量 |

- 影响因素：模型复杂度（参数量/多项式阶数）、训练数据量。

## 最小可运行代码

仓库脚本：`d2l/code/4.4 模型选择_欠拟合和过拟合.py`

```python
import torch
from torch import nn

# 用多项式拟合演示：阶数过高 -> 过拟合，过低 -> 欠拟合
def fit_poly(train_x, train_y, degree):
    net = nn.Linear(degree, 1, bias=False)
    loss = nn.MSELoss()
    opt = torch.optim.SGD(net.parameters(), lr=0.01)
    for _ in range(400):
        opt.zero_grad()
        loss(net(train_x[:, :degree]), train_y).backward()
        opt.step()
    return net
```

## 易错点

- 用测试集调超参 → 泛化误差被泄露，结果虚高。
- 只看训练误差下降就以为模型好，忽视验证误差反弹（过拟合信号）。
- 小数据集单次划分方差大，应用 K 折。

## 复习卡片

- Q: 过拟合的典型特征？
  A: 训练误差很低但泛化误差高（差距大）。
- Q: 为什么不能用测试集选超参？
  A: 会泄露测试信息，导致泛化评估失真。
- Q: 小数据集如何稳健选参？
  A: K 折交叉验证取平均。
