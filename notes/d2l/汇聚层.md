# 6.5 汇聚层 (Pooling)

## 一句话总结

**大白话**：在小窗口里「取个代表」——最大池化取最显眼的那个值，平均池化取平均。这样图变小（省算力）、[[感受野]]变大、还对「物体挪了一两个像素」不敏感。它没有可学习参数，每个通道各算各的。

**严谨说法**：汇聚层在局部窗口取最大值或平均值做下采样，降低空间分辨率、增大[[感受野]]、并对微小平移更鲁棒；它没有可学习参数，逐通道独立进行。

## 本节解决的问题

- 池化解决什么问题（位置敏感性、降采样）？
- 最大池化与平均池化的差异？
- 池化对通道数与梯度有何影响？

## 核心概念 / 公式 / API

| 类型 | 运算 | 特点 |
| --- | --- | --- |
| 最大池化 | 窗口取 max | 保留最显著响应，常用 |
| 平均池化 | 窗口取 mean | 平滑，全局平均池化用于分类头 |

- 无可学习参数；逐通道独立，**输出通道数 = 输入通道数**。
- 提供一定的平移不变性，减小特征图尺寸与计算量。

| API | 说明 |
| --- | --- |
| `nn.MaxPool2d(kernel_size, stride, padding)` | 默认 `stride=kernel_size` |
| `nn.AvgPool2d(...)` | 平均池化 |
| `nn.AdaptiveAvgPool2d(1)` | 全局平均池化到 1×1 |

## 输入输出形状

- `(B,C,H,W) → (B,C,H',W')`，通道不变，尺寸按核与步幅缩小。

## 最小可运行代码

```python
import torch
from torch import nn

X = torch.randn(1, 8, 16, 16)
pool = nn.MaxPool2d(2)              # 2x2, stride=2
print(pool(X).shape)               # (1,8,8,8)

gap = nn.AdaptiveAvgPool2d(1)
print(gap(X).shape)                # (1,8,1,1) 全局平均池化
```

## 易错点

- 误以为 `nn.MaxPool2d` 默认 `stride=1`——其实默认等于 `kernel_size`。
- 池化不改变通道数，别和卷积混淆。
- 过度池化导致空间信息丢失，分割等任务需谨慎。

## 和前后章节 / 真实项目的连接

- 与卷积交替构成经典 CNN 主干 → [[LeNet]]。
- 现代网络多用步幅卷积/全局平均池化替代部分池化 → [[ResNet与DenseNet]]。

## 复习卡片

- Q: 池化层有可学习参数吗？
  A: 没有。
- Q: 池化对通道数的影响？
  A: 不变，逐通道独立。
- Q: `nn.MaxPool2d(2)` 的默认步幅？
  A: 等于 kernel_size（即 2）。
