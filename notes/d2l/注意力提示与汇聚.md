# 10.1-10.2 注意力提示与注意力汇聚 (Nadaraya-Watson)

## 一句话总结

**大白话**：注意力像「带着问题去查资料」。你的问题是查询（query），资料库里每条资料有个标签（key）和内容（value）。拿问题去和每个标签比相似度，越相似的内容给越高权重，最后把内容加权汇总——这就是答案。Nadaraya-Watson 核回归是这套思路最早的雏形。

**严谨说法**：[[注意力机制]]把信息分为「[[查询键值|查询 (query)、键 (key)、值 (value)]]」：用 query 与各 key 的相似度算权重，对 value 加权求和得到输出；Nadaraya-Watson 核回归是其最早的非参数雏形。

## 本节解决的问题

- 注意力的 query/key/value 框架是什么？

- 核回归如何引出可学习的注意力汇聚？

## 核心概念 / 公式 / API

### QKV 框架
- **非自主性提示**：显著刺激（如池化、全连接，被动选）。
- **自主性提示 (query)**：带目标的主动查询。
- 注意力汇聚：

$$f(q)=\sum_i \alpha(q,k_i)\,v_i,\quad \alpha(q,k_i)=\mathrm{softmax}(\text{score}(q,k_i))$$

### Nadaraya-Watson 核回归
- 用高斯核度量 query 与 key 距离作权重：

$$f(x)=\sum_i \frac{\exp(-\frac{1}{2}(x-x_i)^2)}{\sum_j\exp(-\frac{1}{2}(x-x_j)^2)}\,y_i$$

- 加入可学习参数 w 即「带参数注意力汇聚」，是现代注意力的起点。

## 最小可运行代码

```python
import torch

# 非参数 Nadaraya-Watson 核回归
def nw_kernel(queries, keys, values):
    # queries:(n,) keys/values:(m,)
    scores = -((queries[:, None] - keys[None, :]) ** 2) / 2
    weights = torch.softmax(scores, dim=1)     # (n, m) 注意力权重
    return weights @ values                    # 加权求和
```

## 易错点

- 混淆 key 与 value：key 用于算相似度，value 才是被聚合的内容。
- 权重未经 softmax 归一化导致输出尺度异常。

## 复习卡片

- Q: 注意力的三要素？
  A: 查询 query、键 key、值 value。
- Q: 注意力输出的计算式？
  A: 对 value 按 query-key 相似度 softmax 权重加权求和。
- Q: Nadaraya-Watson 用什么度量相似度？
  A: 高斯核（距离的负指数）。
