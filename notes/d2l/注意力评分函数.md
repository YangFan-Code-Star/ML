# 10.3 注意力评分函数

## 一句话总结

**大白话**：评分函数就是「怎么算问题和标签像不像」。两种主流算法：加性注意力用一个小网络来比（query 和 key 维度不同也行）；缩放点积直接做点积、再除以 $\sqrt{d}$（维度一高点积会变很大，除一下防止 softmax 卡死）。后者是 Transformer 的核心。

**严谨说法**：评分函数衡量 [[查询键值|query 与 key]] 的相似度：**加性注意力**用一层 MLP 适配不同维度，**缩放点积注意力**直接做点积并除以 $\sqrt{d}$ 防止数值过大，后者是 Transformer 的核心。

## 本节解决的问题

- query 与 key 维度不同/相同时各用什么评分？
- 为什么点积要除以 $\sqrt{d}$？
- 如何用掩码屏蔽无效位置（padding）？

## 核心概念 / 公式 / API

| 评分函数 | 公式 | 适用 |
| --- | --- | --- |
| 加性注意力 | $\mathbf{w}_v^\top\tanh(\mathbf{W}_q\mathbf{q}+\mathbf{W}_k\mathbf{k})$ | q,k 维度可不同 |
| 缩放点积 | $\dfrac{\mathbf{q}^\top\mathbf{k}}{\sqrt{d}}$ | q,k 同维，计算高效 |

- 除以 $\sqrt{d}$：高维点积方差随 d 增大，缩放后保持 softmax 梯度稳定。
- **掩码 softmax**：对填充位置赋极小值，使其权重→0。

## 输入输出形状

- queries `(B, n, d)`、keys `(B, m, d)` → 注意力权重 `(B, n, m)` → 输出 `(B, n, v)`。

## 最小可运行代码

```python
import torch, math
from torch import nn

class DotProductAttention(nn.Module):
    def __init__(self, dropout):
        super().__init__()
        self.dropout = nn.Dropout(dropout)
    def forward(self, queries, keys, values):
        d = queries.shape[-1]
        scores = queries @ keys.transpose(1, 2) / math.sqrt(d)  # 缩放
        self.weights = torch.softmax(scores, dim=-1)
        return self.dropout(self.weights) @ values
```

## 易错点

- 点积不缩放 → 大维度下 softmax 饱和、梯度消失。
- 掩码用 0 而非 `-inf`（或极大负数）→ softmax 后仍有非零权重。
- 转置维度搞错导致 `(n,m)` 权重矩阵形状不对。

## 复习卡片

- Q: 两种主流评分函数？
  A: 加性注意力与缩放点积注意力。
- Q: 为什么点积除以 $\sqrt{d}$？
  A: 抵消高维点积方差增大，稳定 softmax 梯度。
- Q: 如何屏蔽填充位置？
  A: 掩码处赋极大负值再做 softmax。
