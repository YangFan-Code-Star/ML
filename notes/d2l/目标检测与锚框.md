# 13.3-13.4 目标检测、边界框与锚框

## 一句话总结

**大白话**：分类只回答「这是什么」，检测还要回答「在哪、有几个」——既画框（定位）又贴标签（分类）。做法是先在图上密密麻麻撒一堆预设候选框（锚框），用「框重叠多少」（IoU）判断哪些框框住了真目标，让它们去学类别和位置微调；最后用 NMS 去掉一堆重复框。

**严谨说法**：目标检测要同时定位（边界框）和分类多个物体；[[锚框]]是预设的候选框，通过 [[交并比IoU|IoU]] 与真实框匹配来分配类别与偏移标签，是单阶段/两阶段检测器的共同基础。

## 本节解决的问题

- 边界框如何表示？
- 锚框是什么，为何要预设大量锚框？
- 如何用 IoU 给锚框分配训练标签？

## 核心概念 / 公式 / API

| 概念 | 说明 |
| --- | --- |
| 边界框 | `(x1,y1,x2,y2)` 或 `(cx,cy,w,h)` 两种表示 |
| 锚框 | 在每个像素生成多尺度/多宽高比的候选框 |
| IoU | 交并比 $\dfrac{\lvert A\cap B\rvert}{\lvert A\cup B\rvert}$，衡量框重叠 |
| 偏移量 | 锚框到真实框的回归目标 |
| NMS | 非极大值抑制，去除高度重叠的冗余预测框 |

### 标签分配
- 每个锚框与各真实框算 IoU；IoU 高的判为正样本（学类别 + 偏移），低的为背景。

## 最小可运行代码

```python
import torch

def box_iou(boxes1, boxes2):
    # boxes:(n,4) (x1,y1,x2,y2)
    area1 = (boxes1[:, 2] - boxes1[:, 0]) * (boxes1[:, 3] - boxes1[:, 1])
    area2 = (boxes2[:, 2] - boxes2[:, 0]) * (boxes2[:, 3] - boxes2[:, 1])
    lt = torch.max(boxes1[:, None, :2], boxes2[:, :2])
    rb = torch.min(boxes1[:, None, 2:], boxes2[:, 2:])
    inter = (rb - lt).clamp(min=0).prod(dim=2)
    return inter / (area1[:, None] + area2 - inter)
```

## 易错点

- 边界框两种表示混用导致坐标错乱。
- 锚框数量巨大，正负样本极不平衡，需采样/加权。
- 预测后忘记 NMS → 同一物体多框重复输出。

## 复习卡片

- Q: IoU 衡量什么？
  A: 两个框的交并比（重叠程度）。
- Q: 锚框训练标签如何分配？
  A: 按与真实框的 IoU 判正/负，正样本学类别与偏移。
- Q: NMS 的作用？
  A: 抑制冗余重叠框，保留最优预测。
