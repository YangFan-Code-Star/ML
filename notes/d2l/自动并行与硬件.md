# 12.3-12.4 自动并行与硬件

## 一句话总结

**大白话**：互不相干的活儿可以同时干——框架会自动让「算」和「搬数据」重叠起来省时间。另外要懂硬件：CPU 是几个全能高手（擅长复杂逻辑），GPU 是一大群只会算的小工（擅长海量矩阵运算）；而且很多时候慢不是因为算力不够，而是「数据搬不过来」（带宽瓶颈）。

**严谨说法**：框架能自动并行计算与通信（在多 GPU 间重叠 kernel 执行与数据传输）；理解 CPU/GPU 内存层级、带宽与算力瓶颈，才能写出高效训练代码。

## 本节解决的问题

- 计算与通信如何重叠以提速？
- CPU vs GPU 的架构差异？
- 训练为何常受内存带宽而非算力限制？

## 核心概念 / 公式 / API

### 自动并行
- 不同设备上的独立运算可并行；计算与数据搬运 (`copy`) 可重叠，框架自动调度。

### 硬件要点
| 部件 | 关注点 |
| --- | --- |
| CPU | 少量强核、大缓存，擅长控制流 |
| GPU | 海量弱核，擅长大规模并行（矩阵运算） |
| 内存层级 | 寄存器 > 缓存 > 显存 > 主存，越近越快越小 |
| 带宽 | 很多负载受访存带宽限制，而非峰值算力 |

- 提升利用率：增大 batch、减少 CPU↔GPU 拷贝、用合适数据类型（混合精度）。

## 最小可运行代码

```python
import torch

# 在两个流上并行运算（概念示意）
a = torch.randn(4000, 4000, device="cuda:0")
b = torch.randn(4000, 4000, device="cuda:0")
# 独立运算可被框架并行调度
ya = a @ a
yb = b @ b
torch.cuda.synchronize()
```

## 易错点

- 频繁小 kernel + 同步阻断并行，吞吐低。
- 误以为只要堆算力就快——带宽/拷贝常是真正瓶颈。
- 单机多卡数据搬运频繁会抵消并行收益。

## 复习卡片

- Q: CPU 与 GPU 的核心差异？
  A: CPU 少量强核擅控制流，GPU 海量弱核擅大规模并行。
- Q: 训练常见的真正瓶颈？
  A: 内存带宽与数据搬运，而非峰值算力。
- Q: 如何重叠以提速？
  A: 让独立计算与通信/拷贝并行执行。
