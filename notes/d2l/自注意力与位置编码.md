# 10.6 自注意力与位置编码

## 一句话总结

**大白话**：[[自注意力]]就是「让句子里每个词都直接和其他所有词打招呼」——不管隔多远，一步就能互相关注（不像 RNN 要一个个传），而且能并行算。代价是它天生不知道词的先后顺序（打乱了也一样），所以要额外给每个位置盖一个「位置邮戳」（[[位置编码]]）告诉它谁在前谁在后。

**严谨说法**：自注意力让序列每个位置同时作 query/key/value 互相关注，O(1) 路径捕获任意距离依赖且高度并行；但它本身对顺序不敏感，需用**[[位置编码]]**注入序列位置信息。

## 本节解决的问题

- 自注意力相比 CNN/RNN 的计算复杂度与并行性？
- 为什么自注意力需要位置编码？
- 正弦位置编码如何工作？

## 核心概念 / 公式 / API

### 三种架构对比
| 架构 | 最长路径 | 并行性 | 每层复杂度 |
| --- | --- | --- | --- |
| CNN | O(n/k) | 高 | O(k·n·d²) |
| RNN | O(n) | 低（时序依赖） | O(n·d²) |
| 自注意力 | O(1) | 高 | O(n²·d) |

- 自注意力任意两位置直接交互（最长路径 O(1)），利于长程依赖，但序列长时 O(n²) 开销大。

### 位置编码
- 自注意力对置换等变（打乱顺序结果同样打乱），无序列概念 → 加位置编码。
- 正弦/余弦位置编码：

$$PE_{(pos,2i)}=\sin\!\Big(\frac{pos}{10000^{2i/d}}\Big),\quad PE_{(pos,2i+1)}=\cos\!\Big(\frac{pos}{10000^{2i/d}}\Big)$$

- 不同频率组合编码绝对位置，且能表达相对位置。

## 最小可运行代码

```python
import torch, math
from torch import nn

class PositionalEncoding(nn.Module):
    def __init__(self, d, max_len=1000):
        super().__init__()
        self.P = torch.zeros(1, max_len, d)
        pos = torch.arange(max_len).reshape(-1, 1)
        div = torch.pow(10000, torch.arange(0, d, 2) / d)
        self.P[:, :, 0::2] = torch.sin(pos / div)
        self.P[:, :, 1::2] = torch.cos(pos / div)
    def forward(self, X):
        return X + self.P[:, :X.shape[1], :].to(X.device)
```

## 易错点

- 忘记加位置编码 → 模型分不清词序，性能崩。
- 自注意力 O(n²) 显存/算力，长序列要注意（稀疏/线性注意力等改进）。
- 位置编码维度需与 embedding 维度一致才能相加。

## 复习卡片

- Q: 自注意力相对 RNN 的最大优势？
  A: 最长依赖路径 O(1) 且高度并行。
- Q: 为什么需要位置编码？
  A: 自注意力对顺序不敏感（置换等变），需注入位置信息。
- Q: 自注意力的复杂度瓶颈？
  A: 序列长度的平方 O(n²·d)。
