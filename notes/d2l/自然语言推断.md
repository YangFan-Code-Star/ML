# 15.4-15.5 自然语言推断与注意力

## 一句话总结

**大白话**：给两句话——前提和假设，判断它们是「能推出来（蕴含）/互相矛盾/没关系（中立）」。可分解注意力模型分三步：先用[[注意力机制|注意力]]把两句的词互相「对齐」（谁对应谁），再逐对「比较」，最后「汇总」出结论。不用 RNN，又快又轻。

**严谨说法**：自然语言推断 (NLI) 判断「前提」与「假设」的关系（蕴含/矛盾/中立）；可分解注意力模型用词级软对齐 (attend → compare → aggregate) 在不依赖循环的情况下高效完成句子对匹配。

## 本节解决的问题

- 句子对分类任务如何建模两句关系？
- 如何用注意力做词级软对齐？
- 可分解注意力的三步框架？

## 核心概念 / 公式 / API

### NLI 任务
- 输入：前提 + 假设两句；输出：蕴含 / 矛盾 / 中立（3 分类）。
- 数据集如 SNLI。

### 可分解注意力（attend-compare-aggregate）
| 步骤 | 做法 |
| --- | --- |
| Attend | 用注意力在两句词之间软对齐，得到加权表示 |
| Compare | 把每个词与其对齐内容拼接，经 MLP 比较 |
| Aggregate | 求和/池化两句比较结果，分类 |

- 不用 RNN，仅靠注意力 + MLP，参数少、可并行。

## 最小可运行代码

```python
import torch
import torch.nn.functional as F

def attend(A, B, f):
    # A:(B,la,d) B:(B,lb,d)  f: MLP
    e = f(A) @ f(B).transpose(1, 2)       # 注意力分数 (B,la,lb)
    beta = F.softmax(e, dim=2) @ B        # A 中每词对齐到 B 的软表示
    alpha = F.softmax(e, dim=1).transpose(1, 2) @ A
    return beta, alpha
```

## 易错点

- 句子对要正确拼接/区分（段标记），别混淆前提与假设。
- 注意力 softmax 的维度方向（对 A→B 与 B→A 分别归一化）。
- 类别是 3 分类，别误设 2。
