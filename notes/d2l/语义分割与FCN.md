# 13.9-13.11 语义分割、转置卷积与全卷积网络 (FCN)

## 一句话总结

**大白话**：分类是「整张图是什么」，语义分割是「每个像素分别是什么」（给图上色：这片是路、那片是人）。做法：先用 [[卷积]] 把图缩小提特征，再用「反着的卷积」（转置卷积）把它放大回原尺寸，逐像素出类别。

**严谨说法**：语义分割对每个像素分类；FCN 用卷积骨干下采样提特征，再用**转置卷积**上采样回原分辨率做逐像素预测，是稠密预测任务的基础架构。

## 本节解决的问题

- 语义分割与目标检测的区别？
- 转置卷积如何实现上采样？
- FCN 如何输出与输入同尺寸的预测？

## 核心概念 / 公式 / API

| 概念 | 说明 |
| --- | --- |
| 语义分割 | 逐像素分类（同类不区分实例） |
| 转置卷积 | 可学习的上采样，放大特征图尺寸 |
| FCN | 全卷积，无全连接；末端 1×1 卷积出类别通道 + 转置卷积上采样 |
| 双线性初始化 | 转置卷积常用双线性插值初始化 |

- 输出形状 `(B, num_classes, H, W)`，对每个像素做 softmax 分类。

## 最小可运行代码

```python
import torch
from torch import nn

# 转置卷积上采样：尺寸放大
deconv = nn.ConvTranspose2d(3, 3, kernel_size=4, padding=1, stride=2)
x = torch.randn(1, 3, 16, 16)
print(deconv(x).shape)        # (1, 3, 32, 32)  放大 2 倍

# FCN 末端：1x1 卷积出类别 + 转置卷积上采样到原图
head = nn.Sequential(
    nn.Conv2d(512, 21, 1),                                  # 21 类
    nn.ConvTranspose2d(21, 21, 64, padding=16, stride=32),  # 上采样回原尺寸
)
```

## 易错点

- 转置卷积参数不当导致输出尺寸对不上，需按公式核算。
- 转置卷积可能产生棋盘伪影，可用插值 + 卷积替代。
- 分割标签是像素级整数图，损失对每个像素算交叉熵。

## 复习卡片

- Q: 语义分割的预测粒度？
  A: 每个像素一个类别。
- Q: 转置卷积的作用？
  A: 可学习的上采样，放大特征图分辨率。
- Q: FCN 为什么叫「全卷积」？
  A: 没有全连接层，全部用卷积/转置卷积，可适配任意尺寸输入。
