# 5.5 读写文件（保存与加载）

## 一句话总结

**大白话**：训练好的模型要存盘，存的是「旋钮数值表」（`state_dict`）而不是整台机器——下次先照图纸把机器搭好，再把数值灌回去。这样换环境、换代码版本都不容易出错，也是断点续训和上线部署的基础。

**严谨说法**：用 `torch.save/load` 持久化[[张量]]；模型推荐保存 `state_dict`（参数字典）而非整个模型对象，加载时先构建同结构网络再 `load_state_dict`，这是断点续训与部署的基础。

## 本节解决的问题

- 如何保存/加载张量和字典？
- 保存整个模型 vs 只存 `state_dict` 的取舍？
- 如何恢复模型继续训练或推理？

## 核心概念 / 公式 / API

| 操作 | API |
| --- | --- |
| 存/取张量 | `torch.save(x, "x.pt")` / `torch.load("x.pt")` |
| 存参数 | `torch.save(net.state_dict(), "m.params")` |
| 取参数 | `net.load_state_dict(torch.load("m.params"))` |

- 推荐存 `state_dict`：与代码解耦、跨环境更稳健；存整个对象依赖类定义与路径。

## 最小可运行代码

```python
import torch
from torch import nn

class MLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.hidden = nn.Linear(20, 256)
        self.out = nn.Linear(256, 10)
    def forward(self, X):
        return self.out(torch.relu(self.hidden(X)))

net = MLP()
torch.save(net.state_dict(), "mlp.params")   # 保存参数

clone = MLP()                                 # 先建同结构
clone.load_state_dict(torch.load("mlp.params"))
clone.eval()                                  # 推理前切换模式
```

## 易错点

- 加载 `state_dict` 前必须先构建结构一致的网络。
- 直接 `torch.save(net)` 在不同代码版本/路径下可能反序列化失败。
- 加载后忘记 `eval()`，dropout/BN 仍处训练态。

## 和前后章节 / 真实项目的连接

- 参数即 [[参数管理]] 中的 `state_dict`。
- 真实训练的 checkpoint 还应保存优化器状态与 epoch，便于断点续训。

## 复习卡片

- Q: 推荐保存模型的什么？
  A: `state_dict`（参数字典），而非整个对象。
- Q: 加载参数前要做什么？
  A: 先实例化结构相同的网络。
- Q: 推理前要切换什么模式？
  A: `net.eval()`。
