# 13.12 风格迁移 (Style Transfer)

## 一句话总结

**大白话**：把「这张照片的内容」配上「那幅画的笔触风格」。用预训练 CNN 当裁判：深层特征像不像内容图（内容损失）、纹理统计（Gram 矩阵）像不像风格图（风格损失）。然后——注意——不是训网络，而是直接把「合成图本身」当作要优化的变量，一像素一像素地[[梯度|梯度下降]]改它，直到两个损失都小。

**严谨说法**：神经风格迁移把「内容图的结构」与「风格图的纹理」融合：用预训练 CNN 的不同层特征定义内容损失与风格损失（基于 Gram 矩阵），直接对合成图像的像素做梯度下降优化。

## 本节解决的问题

- 如何分别度量「内容」和「风格」？
- 优化的对象为什么是图像像素而非网络参数？
- Gram 矩阵为何能表征风格？

## 核心概念 / 公式 / API

| 损失 | 定义 |
| --- | --- |
| 内容损失 | 合成图与内容图在某（较深）层特征的 MSE |
| 风格损失 | 合成图与风格图在多层特征的 **Gram 矩阵** 之差 |
| 全变分损失 | 相邻像素差，去噪平滑 |

- **Gram 矩阵** $G=F F^\top$：特征通道间相关性，丢弃空间位置，刻画纹理/风格。
- 优化变量是**合成图像的像素**，网络权重冻结，用梯度下降迭代更新图像。

## 最小可运行代码

```python
import torch

def gram(features):                 # features:(C, H*W)
    n = features.shape[1]
    return features @ features.t() / n     # 通道相关性

# 合成图作为可学习参数，冻结预训练 VGG
synth = torch.nn.Parameter(content_img.clone())
opt = torch.optim.Adam([synth], lr=0.3)
# loss = content_loss + style_weight * style_loss + tv_weight * tv_loss
```

## 易错点

- 误把网络权重当优化对象——这里优化的是图像本身。
- 内容用深层特征、风格用多层 Gram，层选择影响效果。
- 各损失权重需平衡，否则偏内容或偏风格。

## 复习卡片

- Q: 风格迁移优化的对象是什么？
  A: 合成图像的像素，网络权重冻结。
- Q: 风格用什么表征？
  A: 特征的 Gram 矩阵（通道相关性）。
- Q: 内容损失如何定义？
  A: 合成图与内容图在某深层特征的 MSE。
